# 3.3 节图占位与绘制说明

本文档用于说明 `3.3 面向网络行为的 P4 意图感知与种子生成技术` 中各图的占位位置、推荐图名以及后续绘制方式，便于统一出图。以下说明以论文表达为主，并尽量使用本系统已经验证过的场景作为示例。

---

## 图3-3 P4-BISG 总体技术框架

### 图名
图3-3 P4-BISG 总体技术框架

### 放置位置
对应小节：`3.3` 开头与 `3.3.1 技术框架与总体流程`

### 建议画法
建议采用横向总体框图，包含四个模块：

1. 用户意图输入
2. 程序上下文构建
3. 多智能体协同生成
4. testcase 输出

### 推荐内容
- 左侧：用户完整意图，例如“内部主机主动发起连接是否允许”
- 中间左：控制流图、编译产物、拓扑和源码共同构成程序上下文
- 中间右：任务规约、数据包生成、规则生成、oracle 预测、一致性校验
- 右侧：testcase 文件与 run 索引

### 绘图重点
这张图应突出“从意图到 testcase”的完整链路，而不是模块实现细节。

## 图3-4 用户意图到任务规约的转换过程

### 图名
图3-4 用户意图到任务规约的转换过程

### 放置位置
对应小节：`3.3.2 面向网络行为的意图建模与任务规约`

### 建议画法
建议采用自上而下的分层规约图，体现自然语言意图逐步收敛为结构化任务的过程。

### 推荐层次
1. 原始用户意图
2. 角色绑定与意图分类
3. 场景约束、外部动作与观测要求
4. 任务规约输出

### 推荐示例
可使用 Stateful Firewall 作为图中示例：
- 原始意图：内部发起允许，外部发起拒绝
- 角色绑定：internal = h1/h2，external = h3/h4
- 场景约束：正向场景与负向场景
- 外部动作：无或最小规则写入
- 输出：结构化任务规约

## 图3-5 最终 testcase 结构与执行语义示意图

### 图名
图3-5 最终 testcase 结构与执行语义示意图

### 放置位置
对应小节：`3.3.4 测试种子的结构化表示与执行语义`

### 建议画法
建议采用中心对象发散图或组合结构图。中心为 testcase，周围给出五个组成部分。

### 推荐组成部分
1. `packet_sequence`
2. `entities`
3. `control_plane_sequence`
4. `execution_sequence`
5. `oracle_prediction`

### 推荐示例
可在图旁边附一个 Fast Reroute 的简短例子：
- 先执行链路断开
- 再发送后续流量
- 最后根据 oracle 判断是否切换到备份路径

### 绘图重点
该图应突出 testcase 是“具备执行语义的完整测试工件”，而不是单一报文集合。

---

## 统一绘图建议

1. 使用黑白或灰蓝配色，保持学术论文风格。
2. 统一采用矩形框和箭头，不使用过多装饰。
3. 图中文字以中文说明为主，仅保留少量结构化对象名。
4. 图下注释只说明图中符号或缩写，不展开具体实现细节。
