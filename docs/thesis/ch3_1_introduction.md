# 3.1 引言

P4 程序测试的难点，不仅在于如何构造语法合法的数据包，还在于如何围绕目标网络行为组织完整测试场景。对许多具有状态依赖的程序而言，测试结果往往同时取决于输入报文、角色关系、控制条件、执行顺序以及必要的外部事件。若缺少对这些因素的统一组织，测试过程通常只能停留在浅层解析与局部转发行为上，难以触达真正与目标行为相关的关键逻辑。

现有测试方法多围绕路径覆盖、符号约束求解或输入空间扩展展开，能够为程序生成一组结构上合法的输入，但较少直接面向用户提出的行为级测试目标。用户在实际测试中更关注的是某种通信关系是否成立、某类状态是否会在特定条件下发生转移、某种观测结果是否与预期一致，而不是某一条程序路径是否可达。由此可见，P4 测试所需要的并不仅是输入报文本身，还包括与测试目标一致的角色绑定、规则条件、执行动作以及结果判断依据。

围绕这一问题，本章研究面向网络行为的 P4 意图感知与种子生成技术。该技术以用户给出的自然语言测试意图为起点，在程序上下文约束下自动生成结构化测试种子，并使生成结果能够服务于后续状态验证、缺陷触发和真实环境回放。下面先给出相关研究背景与现有方法边界，再逐步引出本章的问题定义、方法设计和实验结果。
