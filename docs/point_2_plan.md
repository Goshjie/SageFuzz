# 第二创新点实验推进计划

## 1. 目标

围绕“面向角色异质性的异构模型分工机制”推进第二创新点，核心目标不是简单替换某个模型，而是验证：

- 结构化审查型节点是否可以安全地下沉到小模型；
- 该分工方案是否优于“所有节点统一切换为小模型”的粗粒度方案；
- 在确定性 validator 保护下，系统是否仍能保持可用性、稳定性与较好的工程性。

当前默认主方案为：

- 保持 `Agent1`、`Agent2`、`Agent3`、`Agent4`、`Agent6` 使用原有大模型；
- 将 `Agent5` 作为首个小模型下沉对象；
- 后续以“全小模型”作为对照实验组。

## 2. 为什么先做 Agent5

选择 `Agent5` 的原因如下：

1. `Agent5` 是结构化审查节点，输入边界最清晰，输入由 `TaskSpec`、单场景 `packet_sequence`、`RuleSetCandidate` 组成，输出仅为 `CriticResult`。
2. `Agent5` 属于判别型任务，不是开放式生成任务，对模型的发散推理要求较低，更适合作为小模型替换的起点。
3. `Agent5` 下游存在强确定性校验：
   - `validate_control_plane_entities(...)`
   - `validate_execution_sequence(...)`
   即使小模型判断出现偏差，系统仍能由 deterministic validator 做二次兜底。
4. 若实体阶段整体失败，系统还有 deterministic fallback，可避免主流程完全失效。
5. 从论文论证角度，`Agent5` 最容易形成可解释结论：成功时说明结构化审查节点适合小模型下沉；失败时也容易定位原因，而不会与上游开放式生成误差混杂。

## 3. 为什么当前先选 Qwen2.5-7B-Instruct

当前阶段优先选择 `Qwen2.5-7B-Instruct` 作为启动小模型，而不是直接从更激进的更小模型或更新代际模型开始，主要原因如下：

1. 当前首要目标是先验证“小模型承担 Agent5 是否可行”，而不是立即追求最小参数量。
2. `Agent5` 任务对严格 JSON 输出、中文/英文混合上下文理解、结构化审查反馈稳定性要求较高；`Qwen2.5-7B-Instruct` 更适合作为稳妥启动点。
3. 在现有 `Agno + OpenAI-compatible` 接口下，`Qwen2.5-7B-Instruct` 已完成独立接入验证，能够稳定输出 JSON，并已在真实 `Agent5` prompt 上完成正负样本初步验证。
4. 待 `Agent5` 小模型路线跑通后，再进一步测试更小或更新代际的模型，例如 `Qwen3` / `Qwen3.5`，会更利于控制变量。

因此，本阶段将 `Qwen2.5-7B-Instruct` 视为“启动验证模型”，而不是最终唯一候选模型。

## 4. 代码与分支推进策略

### 4.1 基线提交

先对当前系统做一次基线提交，保留当前大模型版本与论文材料状态，作为后续实验的可回退基线。

### 4.2 分支规划

建议使用以下分支：

- `feat/agent-model-overrides`
  - 用于实现“按 agent 覆盖模型配置”的基础能力。
- `exp/agent5-small-model`
  - 用于“仅 Agent5 使用小模型”的主实验。
- `exp/all-small-model`
  - 用于“所有 agent 统一切换小模型”的对照实验。
- `exp/agent5-finetune-prep`
  - 用于后续如需构建微调数据、对接 LoRA / QLoRA 的探索。

日常开发先在 `feat/agent-model-overrides` 上完成基础能力，再按实验内容切换到对应分支推进。

## 5. 实施顺序

### 阶段一：小模型独立验证（已启动）

目标：先确认小模型能够承担 `Agent5` 的结构化审查任务。

当前已完成：

- 远端小模型 API 接通；
- `Agno` 下最小 JSON 输出验证通过；
- 使用真实 `Agent5` prompt 和真实历史样本完成正样本 / 负样本初步验证。

### 阶段二：实现按 agent 覆盖模型配置

目标：在不破坏现有 workflow 的前提下，实现：

- 默认沿用原大模型配置；
- 可单独为 `Agent5` 指定另一套模型配置；
- 可选支持“全小模型”一键切换，作为对照组。

### 阶段三：小规模真实任务实验

先选 3 个代表任务：

- Stateful Firewall
- Link Monitor
- Fast Reroute

分别比较三组配置：

1. 全大模型；
2. 仅 `Agent5` 小模型；
3. 全小模型。

记录：

- testcase 最终生成成功率；
- `Agent5` 与 deterministic validator 一致率；
- 各阶段重试次数；
- fallback 触发率；
- 总耗时。

### 阶段四：论文实验扩展

若小规模实验结果支持当前假设，再扩展到 5 个程序 × 多个 intent 的正式实验集，并整理图表与误差分析。

## 6. 论文中如何表达

该点不表述为“将某个 agent 换成小模型”，而表述为：

> 面向角色异质性的异构模型分工机制。

具体含义为：

- 开放式生成节点继续使用大模型；
- 结构化审查节点下沉到小模型；
- 借助确定性 validator 维持整体正确性；
- 通过与“全小模型”方案的对照实验，说明并非所有节点都适合统一下沉。

因此，实验现象只是验证手段，真正的工作点在于：

1. 对多智能体节点任务异质性的分析；
2. 对合理小模型落点的选择；
3. 对异构分工策略的实现与验证。
